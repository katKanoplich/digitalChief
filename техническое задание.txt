Технические требования для бизнес-модели с двумя сущностями, связанными отношением "один ко многим":

Предметная область: Книжный магазин

Сущности:

Книга (Book)
Книжный магазин (BookStore)
Отношение: Один книжный магазин может иметь множество книг, но каждая книга принадлежит только одному книжному магазину.

Описание сущностей:

1.Книга (Book):

    id - Идентификатор книги
    title -Название книги
    author - Автор книги
    publisher - Издательство
    year - Год издания
    genre - Жанр
    price - Цена
    quantity - Количество на складе
    bookStore - Принадлежность к книжному магазину

2.Книжный магазин (BookStore):

    id - Идентификатор книжного магазина
    name - Название книжного магазина
    address - Адрес книжного магазина
    contact - Контактная информация
    books - Список книг, принадлежащих данному магазину

Требования:
    1.Реализовать возможность добавления, удаления, редактирования и поиска по id книг.
    2.Реализовать возможность добавления, удаления, редактирования и поиска по id книжных магазинов.

Пример представления данных в JSON формате:
{
    "title": "Book Title",
    "genre": "Fiction",
    "author": "Author Name",
    "publisher": "Publisher Name",
    "price": 19.99,
    "year": 2022,
    "quantity": 10,
    "bookStore":
    {
        "id": 1,
        "name": "Main Street Books",
        "address": "123 Main St, Grmany",
        "contact": "455-1234"
    }
}

    Предполагается использование RESTful API для взаимодействия с системой.
    Доступные HTTP-методы: GET, POST, PUT, DELETE.
    Формат данных: JSON.

    Технологии и инструменты
- Платформа: Spring Boot
- База данных: PostgreSQL
- Язык программирования: Java
- Инструменты разработки: IntelliJ IDEA, Git, GitHub, Maven, hibernate.

  Этапы разработки
1.Проектирование и моделирование сущностей "Книша" и "Книжный магазин".
2.Реализация REST-контроллеров для управления данными.
3.Интеграция с базой данных PostgreSQL.
4.Разработка бизнес-логики для взаимодействия с сущностями.
5.Реализация API-методов для выполнения CRUD-операций.
6.Тестирование и отладка системы.
7.Развертывание приложения и документирование API.
